quippy.clusters.create_hybrid_weights(at, args_str[, error])¶

Given an atoms structure with a hybrid_mark property, set to HYBRID_ACTIVE_MARK for some atoms, this routine adds transition and buffer regions, and creates a weight_region1 property, whose values are between 0 and 1.

Atoms marked with HYBRID_ACTIVE_MARK in hybrid_mark get weight 1. Neighbour hopping is done up to transition_hops times, over which the weight linearly decreases to zero with hop count if weight_interpolation=hop_ramp. If weight_interpolation=distance_ramp, weight is 1 up to distance_ramp_inner_radius, and goes linearly to 0 by distance_ramp_outer_radius, where distance is calculated from distance_ramp_center(1:3). distance_ramp makes the most sense if the HYBRID_ACTIVE_MARK is set on a sphere with radius distance_ramp_inner_radius from distance_ramp_center, no hysteresis, and with transition_hops < 0 so hops continue until no more atoms are added. Transition atoms are marked with HYBRID_TRANS_MARK. If hysteretic_buffer=F (default), buffer atoms are selected by hopping buffer_hops times, otherwise hysteretically with radii hysteretic_buffer_inner_radius and hysteretic_buffer_outer_radius. Buffer atoms are marked with HYBRID_BUFFER_MARK and given weight 0. If hysteretic_connect is set, all hops are done on bonds in the hysteretic_connect structure.

quippy.clusters.carve_cluster(at, args_str, cluster_info[, mark_name, error])¶

Create cluster from atoms and cluster information table (which which should be generated by a call to create_cluster_info_from_hybrid_mark().

The output cluster contains all properties of the initial atoms object, and some additional columns, which are:

index
index of the cluster atoms into the initial atoms object.
termindex
nonzero for termination atoms, and is an index into the cluster atoms specifiying which atom is being terminated, it is used in collecting the forces.
rescale
a real number which for nontermination atoms is 1.0, for termination atoms records the scaling applied to termination bond lengths
shift
the shift of each atom

args_str options

Name Type Default Comments
cluster_calc_connect bool False Run calc_connect before doing stuff (passivation, etc)
cluster_fix_lattice float
Fix the cluster lattice to be cubic with given 3 lengths. useful with DFT codes where one doesn’t pay for vacuum.
cluster_round_lattice float 3.0 Distance to which to round cluster lattice (default is 3.0 A)
cluster_same_lattice bool False Don’t modify cluster cell vectors from incoming cell vectors
cluster_vacuum float 10.0 Amount of vacuum to add around cluster (minimum - lattice vectors are rounded up to integers)
do_rescale_r bool False Do rescale cluster lattice positions and lattice (for matching lattice constants of different models
hysteretic_connect bool False Use hysteretic connection object for identifying bonds
map_into_cell bool True Call map_into_cell (lattice coordinates=[-0.5,0.5)) on cluster
print_clusters bool False Print out clusters into clusters.xyz file (clusters.MPI_RANK.xyz if in MPI)
r_scale float 1.0 Cluster position/lattice rescaling factor
randomise_buffer bool True Randomly perturb positions of buffer atoms
run_suffix str None String to append to names of extra properties added to cluster. NOTE: does not override mark_name argument
terminate bool True Do hydrogen termination

quippy.clusters.create_cluster_info_from_mark(at, args_str[, cut_bonds, mark_name, error])¶

Create a cluster using the mark_name (optional arg, default hybrid_mark) property and options in args_str. All atoms that are marked with anything other than HYBRID_NO_MARK will be included in the cluster; this includes active, transition and buffer atoms. Atoms added (to satisfy heuristics) will be marked in new property modified_//trim(mark_name) as HYBRID_BUFFER_MARK. Returns a Table object (cluster_info) which contains info on atoms whose indices are given in atomlist, possibly with some extras for consistency, and optionally terminated with Hydrogens, that can be used by carve_cluster().

args_str options

Name Type Default Comments
cluster_allow_modification bool True Allow cluster to be modified using various heuristics
cluster_heuristics_nneighb_only bool True Apply is_nearest_neighbor test to bonds when deciding what’s bonded for heuristics. If false, uses connectivity object as is
cluster_hopping bool True Identify cluster region by hopping from a seed atom. Needed so that shifts will be correct.
cluster_hopping_nneighb_only bool True Apply is_nearest_neighbor test to bonds when doing bond hops to find cluster. If false, uses connectivity object as is
cluster_periodic_x bool False Do not add vaccum along x, so as to make cluster’s cell vectors periodic
cluster_periodic_y bool False Do not add vaccum along y, so as to make cluster’s cell vectors periodic
cluster_periodic_z bool False Do not add vaccum along z, so as to make cluster’s cell vectors periodic
cluster_same_lattice bool False Don’t modify cluster cell vectors from incoming cell vectors
even_electrons bool False Remove passivation hydrogen to keep total number of electrons even
hysteretic_connect bool False Use hysteretic connect object to decide what’s bonded. Usually goes with cluster_*_nneighb_only=F.
in_out_in bool False Apply heuristic than adds atoms to avoid IN-OUT-IN configurations. Usually equivalent to reduce_n_cut_bonds.
in_out_in_mode str all Mode to use for in_out_in heuristic - can be either ‘all’ or ‘any’. Default is ‘all’.
terminate bool True Do hydrogen termination for dangling bonds
termination_clash_factor float 1.2 Factor to multiply H-H bond-length by when deciding if two Hs are too close. Default 1.2
termination_rescale float 0.0 Rescale factor for X-H passivation bonds
Parameters:

at : Atoms object

args_str : input string(len=-1)

cut_bonds : Table object, optional

Return a list of the bonds cut when making the cluster. See create_cluster() documentation.

mark_name : input string(len=-1), optional

error : in/output rank-0 array(int,’i’), optional
Returns:
